What This Demo Is Showing

This demo shows how one person can move through real-world services across health care, housing, utilities, pharmacy, insurance, legal support, education, employment, and family services.

Each card is a different organization. The person controls what gets shared, and each organization stays responsible for its own records.

The walkthrough set is grounded in common city pressure points: housing bottlenecks, high referral volume, continuing outreach demand, and document-readiness gaps.

  1. Start with Sovereign's Data. This is the person's home base for records, permissions, and trusted credentials. The data can live in a managed service or be taken with the person and self-hosted so they stay fully in charge. Open Sovereign Solid Pod to see direct record storage, or open Sovereign Credential Manager to see the credential and consent view in a more structured format.
  2. Use Identity and Proof Document VC Issuers to convert common documents (state ID, passport, birth certificate, utility bill, proof of residency, social security) into trusted digital credentials.
  3. Browse by business type: open a 2-digit sector, then open a 3-digit group to see organizations in that area.
  4. Run actions inside a card. Some actions create a handoff to another organization (a referral). Then go to the receiving organization and use Fulfill selected referral to complete the next step.
  5. Each self-service action shows a clear credential requirement check before it runs. Examples: health intake usually needs government ID, utilities and housing usually need ID plus residency proof, and insurance/claims usually need ID plus social security. Safety-first actions still start even when documents are missing.
  6. Use Download Non-Identifiable Audit Log to export an accountability log with masked person references and no source-document data.
  7. Use Reset Demo Data any time to clear everything and start a fresh scenario.

The point is simple: coordinated care without one giant database. Sharing is permission-based, and accountability stays with each organization.
